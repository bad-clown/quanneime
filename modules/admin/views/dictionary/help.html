系统字典配置说明

格式说明

系统字典是由一条一条的记录组成，每条记录包含4个部分：分类索引(Index)、关键字(Key)、值(Value)、描述(Description)，如下所示：

Index Key Value Description
App SystemName iBangbuy 系统名称

分类索引用来区分不同的分类，每个分类中的关键字都不相同，但是不同分类内的关键字可以相同。 关键字-值组成一个键值对，系统中通过分类索引和关键字来获取具体的值。 描述是对这条记录的一个说明，没有实际意义。 通过修改Value值，可以达到动态配置系统的目的。

重要：不要随便删除记录，否则可能导致系统出严重问题。

配置示例

详细配置项说明

如配置示例所示，通过查找对应的Index和Key，可以定位一条配置项，然后对该配置项修改就可以达到配置系统的目的。所以以下列出的配置项列表，都会指定Index和Key。

非常重要：该文档中未提及的字典项，不要修改或删除。

1. 系统相关

1.1 系统名称

Index: App
Key: SystemName

2. 公司类型配置

Inex: CompanyType

可以通过添加条目来增加公司类型，也可以修改已有的Value值来修改公司类型

3. 职位类型配置

Index: PositionType

和公司类型配置类似，可以添加或修改职位类型的Value值，添加时注意Key值不要重复

4. 系统管理员配置

Index: AdminUsers

添加Index为AdminUsers的条目来添加系统管理员，Key和Value值均为要添加的管理员用户名（不是昵称）
